# output/

This directory contains auto-generated Terraform remediation code.

## `remediation.tf`

- **Generated by**: RemediationArchitect agent
- **Overwritten** on each scan cycle — never edit this file manually
- **Purpose**: Contains the consolidated HCL for all remediations proposed in the current run
- **Applied via**: `tflocal apply` (or `terraform apply`) after explicit approval through the Approval Gate

### Lifecycle

1. RemediationArchitect reads `output/findings_store.json` and runs dependency checks
2. For each actionable finding, it generates remediation HCL
3. All remediations are written to `output/remediation.tf`, replacing any previous content
4. The Approval Gate displays the HCL diff and waits for `APPROVE <resource-id>`
5. On approval, the file is submitted to `tflocal apply`

### Important

- Do **not** commit this file to version control in production — it reflects transient infrastructure state
- Do **not** manually edit — your changes will be lost on the next scan
- If `terraform validate` fails on this file, the Approval Gate will block and surface the error
